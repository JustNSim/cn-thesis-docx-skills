# Literature Review DOCX Preflight

Resolve and record these fields before drafting Markdown or creating DOCX.

1. **Template mode**: user template with path; bundled base template; or Markdown only. If the user already chose one, acknowledge it rather than asking again.
2. **Word-count target**: minimum/maximum or target count, and whether cover, TOC, references, captions, and tables are excluded. Do not invent a target.
3. **Degree level**: ask the user to choose one of `本科`, `硕士`, `博士`, or `其他`. Use it to set depth, evidentiary caution, and content richness. Do not infer it from the topic.
4. **Numbering owner**: use template numbering by default. Manual Chinese/Arabic prefixes are forbidden when a template heading style already renders numbering.
5. **Output scope**: Markdown, DOCX, or both; and the requested filenames.

When fields 1, 2, or 3 are absent, ask one short question, for example: “请确认模板模式、正文目标字数或区间，以及学位层次。本科、硕士、博士或其他；参考文献、目录和封面是否计入字数？”

For DOCX, copy the selected template, map content to existing slots, and preserve all non-slot package content. A template choice does not authorize reconstructing the document from a blank body.
